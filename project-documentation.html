Event-driven order processing pipeline with SMS confirmation, built on Azure Functions Python v2. Input from webapp → Blob Storage → Event Grid → Durable Functions orchestration (validate, SMS, race confirm vs timeout) → output to Cosmos DB or Table Storage. 6. Auto-Increment Counter / Contatore (Optimistic Concurrency)

🇬🇧English — The Problem

In a distributed environment with multiple Function instances, a simple counter += 1 causes race conditions:

🇮🇹Italiano — Il Problema

In un ambiente distribuito con più istanze della Function, un semplice counter += 1 causa race condition:

sequenceDiagram participant A as Instance A participant B as Instance B participant DB as Cosmos DB A->>DB: read('orderCounter') B->>DB: read('orderCounter') DB-->>A: {currentValue: 42} DB-->>B: {currentValue: 42} Note over A,B: BOTH read 42! A->>A: newValue = 43 B->>B: newValue = 43 A->>DB: write({currentValue: 43}) B->>DB: write({currentValue: 43}) Note over A,B: BOTH write 43 → DUPLICATE!

🇬🇧English — Solution: Compare-And-Swap with ETag

Cosmos DB assigns a unique _etag to each document, changing on every write. Writes are conditional: "write only if ETag hasn't changed".

🇮🇹Italiano — Soluzione: Compare-And-Swap con ETag

Cosmos DB assegna un _etag univoco a ogni documento, che cambia a ogni scrittura. La scrittura viene condizionata: "scrivi solo se l'ETag non è cambiato".

sequenceDiagram participant A as Instance A participant B as Instance B participant DB as Cosmos DB A->>DB: read('orderCounter') B->>DB: read('orderCounter') DB-->>A: {v:42, _etag:"a"} DB-->>B: {v:42, _etag:"a"} A->>A: newValue = 43 B->>B: newValue = 43 A->>DB: replace({v:43}, etag="a") DB-->>A: OK (v:43, _etag:"b") B->>DB: replace({v:43}, etag="a") DB-->>B: 412 PRECONDITION FAILED (etag changed!) Note over B: RETRY: re-read (v:43, etag:"b") B->>B: newValue = 44 B->>DB: replace({v:44}, etag="b") DB-->>B: OK (v:44, _etag:"c") Note over A,B: Instance A → displayId=43, Instance B → displayId=44. OK!

Implementation / Implementazione

class OrderCounter:
    def __init__(self, container, max_retries: int = 3):
        self.container = container
        self.max_retries = max_retries

    async def get_next_id(self) -> dict:
        for attempt in range(self.max_retries + 1):
            try:
                counter = await self.container.read_item(
                    item="orderCounter", partition_key="counter"
                )
                new_value = counter.get("currentValue", 0) + 1
                await self.container.replace_item(
                    item=counter,
                    body={**counter, "currentValue": new_value},
                    etag=counter["_etag"],
                    match_condition="IfNotModified",
                )
                return {
                    "displayId": new_value,
                    "displayOrder": f"ORD-{new_value:04d}",
                    "instanceId": f"ORD-{new_value:04d}-{uuid4().hex[:8]}"
                }
            except exceptions.CosmosResourceNotFoundError:
                await self.container.create_item(
                    body={"id": "orderCounter", "partitionKey": "counter", "currentValue": 0}
                )
                continue
            except exceptions.CosmosAccessConditionFailedError:
                if attempt == self.max_retries:
                    raise CounterConflictError("Max retries exceeded")
                await asyncio.sleep(0.1 * (2 ** attempt))
                continue
