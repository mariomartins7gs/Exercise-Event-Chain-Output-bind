1 Create Resource Group

📥Input

  • Azure subscription con crediti Azure for Students
  • Location: westeurope
  • RG name: rg-orderprocessing-001

⚙️Process

az group create \
  --name rg-orderprocessing-001 \
  --location westeurope

2 Deploy Bicep — 8 Azure Resources + RBAC

📥Input

  • Resource Group rg-orderprocessing-001 (da Step 1)
  • File: infra/main.bicep + 6 moduli in infra/modules/
  • File: infra/parameters.json

⚙️Process

az deployment group create \
  --resource-group rg-orderprocessing-001 \
  --template-file infra/main.bicep \
  --parameters @infra/parameters.json

Durata: ~5-8 minuti (Cosmos DB free tier è il più lento)

📤Output

      "functionPrincipalId": { "value": "00000000-0000-0000-0000-000000000000" }

✅Verify

  --resource-group rg-orderprocessing-001 \
  --output table

# Verifica gli output del deploy
az deployment group show \
  --resource-group rg-orderprocessing-001 \
  --name main \
  --query properties.outputs

AuthorizationFailedIl Service Principal / utente non ha permessi. az role assignment list --assignee <object-id>

5.3 Conferma l'ordine

📥Input

INSTANCE_ID="ORD-0001-a1b2c3d4"

⚙️Process

curl -X POST \
  https://func-order-001.azurewebsites.net/api/confirm_handler \
  -H "Content-Type: application/json" \
  -d "{\"instance_id\": \"$INSTANCE_ID\"}"

📤Output

HTTP/1.1 200 OK — Order confirmed

Lo status passerà a confirmed → processing → completed

⚠️Common Errors

400 Missing instance_idPayload JSON non contiene instance_id
410 GoneL'orchestrazione è già terminata (expired o completed)
404 Orchestration not foundinstance_id inesistente o errato

5.4 Verifica stato finale

⚙️Poll fino a completed

# Loop di polling (Bash)
while true; do
  STATUS=$(curl -s "https://func-order-001.azurewebsites.net/api/status?orderId=$ORDER_ID" | jq -r .status)
  echo "$(date +%H:%M:%S) — $STATUS"
  [ "$STATUS" = "completed" ] && break
  [ "$STATUS" = "expired" ] && break
  sleep 3
done

📤Expected States (conferma)

submitted validating sms_sent pending_confirmation ← invoca confirm_handler qui confirmed processing completed ← stato terminale ✅

📤Expected States (timeout)

submitted validating sms_sent pending_confirmation expired ← stato terminale (dopo 180s)

⚠️Stuck States

Stuck su "sms_sent"L'evento BlobCreated non ha triggerato l'orchestrazione. Verifica Event Grid subscription
Stuck su "pending_confirmation"Il timer DF non è scattato o il raise_event non è stato ricevuto
Status mai "completed"La scrittura su Cosmos DB fallisce. Verifica RBAC: Cosmos DB Built-in Data Contributor

6 Configure GitHub OIDC (One-Time Setup)

Configurazione necessaria per eseguire la CI/CD pipeline da GitHub Actions.

📥Input

  • GitHub repository: Exercise-Event-Chain-Output-bind
  • Azure subscription ID
  • Azure AD Tenant ID

⚙️Process

# 1. Crea App Registration
CLIENT_ID=$(az ad app create \
  --display-name "github-order-pipeline" \
  --query appId -o tsv)

# 2. Crea Service Principal
az ad sp create --id $CLIENT_ID

# 3. Assegna Contributor alla subscription
az role assignment create \
  --assignee $CLIENT_ID \
  --role Contributor \
  --scope /subscriptions/$(az account show --query id -o tsv)/resourceGroups/rg-orderprocessing-001

# 4. Crea federated credential (GitHub Actions)
#    Azure Portal → App Registration → Certificates & Secrets
#    → Federated credentials → Add credential
#    Scenario: GitHub Actions deploying to Azure
#    Organization: (tuo-github-org)
#    Repository: Exercise-Event-Chain-Output-bind
#    Entity type: Environment → production

📤GitHub Secrets da creare

Repository → Settings → Secrets and variables → Actions → New repository secret:

AZURE_CLIENT_ID:       (appId da step 1)
AZURE_TENANT_ID:       (dal tenant Azure AD)
AZURE_SUBSCRIPTION_ID: (subscription ID)

⚠️Common Errors

Federated credential not foundL'entity type deve corrispondere: usa Environment con nome production
403 — insufficient permissionsIl Service Principal non ha Contributor sul RG
404 — resource group not foundEsegui Step 1 prima di configurare OIDC

⏱ Deploy Quick Reference

# 1. Resource Group
az group create --name rg-orderprocessing-001 --location westeurope

# 2. Bicep (8 risorse + RBAC) — ~5 min
az deployment group create \
  --resource-group rg-orderprocessing-001 \
  --template-file infra/main.bicep \
  --parameters @infra/parameters.json

# 3. Function App — ~2 min
func azure functionapp publish func-order-001 --python

# 4. Webapp — ~30s
az storage blob upload-batch \
  --account-name storder001 \
  --auth-mode login \
  --destination '$web' \
  --source webapp/ \
  --overwrite

# 5. E2E Smoke Test
ORDER_ID=$(curl -s -X POST \
  https://func-order-001.azurewebsites.net/api/submit_order \
  -H "Content-Type: application/json" \
  -d '{"customerName":"Test","customerPhone":"+351911234567","items":[{"itemId":"ITM-001","name":"Laptop","quantity":1,"unitPrice":1200}],"total":1200,"tax":276}' | jq -r .orderId)
echo "Order: $ORDER_ID"

# Poll fino a pending_confirmation
sleep 10
curl -s "https://func-order-001.azurewebsites.net/api/status?orderId=$ORDER_ID" | jq .

# Conferma
curl -X POST \
  https://func-order-001.azurewebsites.net/api/confirm_handler \
  -H "Content-Type: application/json" \
  -d "{\"instance_id\": \"$ORDER_ID\"}"

# Poll fino a completed
while true; do
  STATUS=$(curl -s "https://func-order-001.azurewebsites.net/api/status?orderId=$ORDER_ID" | jq -r .status)
  echo "$STATUS"
  [ "$STATUS" = "completed" ] && break
  [ "$STATUS" = "expired" ] && break
  sleep 3
done
